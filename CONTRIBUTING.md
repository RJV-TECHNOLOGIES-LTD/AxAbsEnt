## ⚙️ Contribution Types

### 1. 📜 Theory Extension

If proposing a **modification or extension** to the theoretical framework:

- Use the issue template: `theory_extension.md`
- Must reference: CEFT, SDI, FDT, or SRI consistency
- Attach a **formal derivation** or mathematical argument
- Proposals must be falsifiable, reproducible, and internally complete

> 📛 *Philosophical interpretations, speculative analogies, or ambiguous metaphors will not be considered.*

---

### 2. 🧮 Code Contributions

- Code must be committed in alignment with the AxAbsEnt directory hierarchy (e.g. `src/axabsent/core/`, `forces/`, `simulation/`, etc.)
- All new modules require:
  - Fully typed Python 3.11+ code
  - Unit tests under `tests/`
  - Documentation (both inline and `docs/api/`)
  - Adherence to `.flake8`, `mypy.ini`, `pyproject.toml`, and `tox.ini` standards

### 3. 🧪 Simulation Modules

- Proposals must describe:
  - The force, tensor field, or entity being simulated
  - Why it belongs in the unified framework
  - How it interfaces with CEFT, SDI, or FDT

---

## 🛠 PR Guidelines

- Fork the repository
- Create a branch using the format: `feature/<name>` or `fix/<name>` or `theory/<name>`
- Follow **semantic commit** standards (enforced via `commitizen`)
- All PRs must pass:
  - ✅ Unit tests
  - ✅ Type checks
  - ✅ Linting (`flake8`, `black`, `isort`)
  - ✅ Ontological review (if applicable)
- Submit using the **pull request template**
