## 🛠️ Reporting a Vulnerability

### 1. 🚨 Standard Software Vulnerabilities

If you discover a technical vulnerability in the code (e.g. buffer overflow, execution break, memory leak):

**Please email:** `security@rjv-technologies.com`  
Use PGP encryption (key available in `/SECURITY-PGP.asc`)  
You may also open a confidential security issue using GitHub's private vulnerability reporting feature.

We will respond within **72 hours** and assign a **CVE ID** if applicable.
